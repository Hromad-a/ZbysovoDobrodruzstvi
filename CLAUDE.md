# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

"Zbýšovo Dobrodružství" — a cozy top-down walking game: walk a black cat on a leash through a Prague-style panelák neighbourhood, meet philosophizing hedgehogs, and reach the person who arrives by tram (2–5 minutes into the game) on the other side. All in-game text and dialogue is in Czech.

## Running

The whole game is `index.html` with inline CSS and JS — no build step, no dependencies to install, no tests. The only dependencies are vendored locally so the game works offline and from `file://` (do not swap them for CDN links): Rough.js 4.6.6 as `rough.min.js`, and the Pangolin font (`Pangolin-Regular.ttf`, OFL-licensed, handwriting style with Czech diacritics) as the primary game font on all platforms.

```
python3 -m http.server 8000
```

or just open `index.html` in a browser. Works on desktop (WASD / arrows) and mobile (floating touch joystick via pointer events).

## Architecture

Everything lives in the single `<script>` block of `index.html`, organized into banner-commented sections in this order:

- **Constants / palette** — `INK`, `PAPER`, `PAL` color palette, world size (`WORLD_W`/`WORLD_H` = 2000×1500), `LEASH_LEN`, `BOIL_MS`.
- **Sprites** — all art is procedurally drawn with Rough.js, in a deliberate sketchy child-drawing / Die Brücke style (this is intentional placeholder art, to be swapped for hand-drawn PNGs later). `makeSprite()` pre-renders each sprite to 2 offscreen canvases with different Rough.js seeds; the renderer alternates between them every `BOIL_MS` for a "boiling line" animation. Rough.js options come from the `opts(seed, fill, style, extra)` helper — seeds are explicit and deterministic so the art is stable frame to frame.
- **Audio** — sampled clips generated with ElevenLabs, vendored as `.mp3` (`meow`, `hiss`, `purr`, `bark`, `thunder`, `rain`, `car`, `tram`, `shout`, `mumble`, `thump`, `step`). The cat mumbles (`mumble`) while talking, hedgehogs stomp (`thump`) while delivering a line, the player has soft footsteps (`step`), cars/tram play as they pass, and an occasional off-screen drunk `shout` makes the cat grumble a street line. `initAudio()` (first click/keypress, autoplay policy) fetches + decodes them into `SFX` buffers via the Web Audio API and starts the looping `rain` bed; `rainGain` tracks `weatherLevel` in `updateAudio()` (per frame from `loop()`); one-shots play through `playClip`. `muted` toggles via the `M` key or the top-right speaker icon. To regenerate a clip, call ElevenLabs `POST /v1/sound-generation` — never commit the API key.
- **World** — static layout data (`BUILDINGS`, `CARS`, `BINS`, `TREES`, `LAMPS`, `GRASS`, `COURTYARDS`, roads, park polygon). The ground is green grass in separate, non-overlapping zones — one `GREENZONES` rectangle per building cluster plus the organic `PARK` polygon — spaced apart with bare ground between them (no green is ever drawn over other green). They're drawn under one canvas clip of (world − road bands − `BUILDINGS`), so grass stays off the roads and each building sits in a clean hole. Every tree is placed inside a zone/park (no per-tree patches). World outlines use high Rough.js `roughness` for a hand-drawn crooked look. Obstacles register AABB colliders via `block()`. The whole world is pre-rendered once into 2 boil-variant canvases (`worldLayers`); the camera just blits a viewport from them. `GRASS` holds large green regions around each building cluster; those plus an auto green patch under every tree are drawn before the roads and buildings (which paint over any overlap), so every tree sits on green and no tree touches a road. Keep trees off the road bands (`HROAD`/`VROAD`) or their patch gets covered.
- **Dialogue** — string arrays (`HEDGE_LINES`, `HEDGE_ALT`, `CAT_LINES`), Czech. Speech bubbles go through `say(ent, text, dur)`.
- **Entities & game state** — `gameState` flows `start → play → ending → done`. On the (transparent-background) select screen a big `SPR.jumpcat` leaps across the canvas behind the UI in quick glimpses (`introCat`/`updateIntro`/`drawIntro`, driven from `loop()`), each jump using a different splayed-leg variant (`JUMP_LEGS`). The cat is a small FSM: `wander` (orbits the player with sniff pauses), `approach → lay` (scripted hedgehog conversation via a timed `cat.script` array; delivering a carried worm swaps in a thank-you script with useful hints), `toWorm → playWorm` (bats a park earthworm around, then carries it; ~35% of the time he later eats it in `eatWorm` and apologizes instead of delivering), `dashTree → inTree` (climbs a nearby tree for ~3s, drawn via `cat.zOff` height offset), and `toDest → petted` for the ending. The leash is a hard distance constraint clamping the cat to `LEASH_LEN` from the player; towing him for ~0.6s while he lounges makes him get up and walk along. After the player walks steadily in one direction for ~2s (`player.moveStreak`), the cat heels — trotting up beside/in front instead of being dragged. When climbing, the cat sprite is flipped by which side of the trunk he's on so his paws grip toward the tree. The player has idle quirks when standing still (`player.special`): the man (boy) checks his `phone`, the woman (girl) crouches to `pet` the nearby cat. Tree climbing stores a per-tree `perch` height in `treeSpot` and lands via a `downTree` state that holds the cat in place until `zOff` reaches the ground (so he never floats off in mid-air). A nearby dog throws the cat into a `scared` state (arched pose, "Fss!") where he bolts to the player and cowers until the dog leaves.

The cat also chatters while wandering (`catChatCooldown`): `CAT_STREET_LINES` grumbling about noisy neighbours when people are near, else quiet `CAT_MUSINGS`. He periodically relieves himself on a nearby patch of grass (`toRelief → relief`, gated by `isGreen()` and leash reach), leaving a `pee`/`poop` decal in `marks`, then `bury`s it with an animated scratch pose (`bury0/bury1`) that flings `dirt` specks.

Ambient life: `peds` are neighbours out walking, drawn with their own `SPR.ped` sprites (rounder head, coat, optional hat; one of the `PED_PALS` clothing/hair palettes per ped) so they read as distinct from the two mains; some have a `dog` (varied `DOG_TYPES`/`scale`, leashed or free); dogs follow their owner and frighten the cat via `nearestDog()`. A dog near the cat barks ("Haf!") and lunges, then the owner (`ped.pullT`) strides away hauling it off. `movingCars` are rare traffic that crosses the horizontal road. `winFolks` are people and cats that briefly appear framed inside real building windows (`ALLWINS` collects every window centre; the `SPR.face.{woman,man,cat}` sprites are window-sized (20×24) and drawn centre-anchored so they sit in an existing window rather than adding a new one; `PEEK` is snapped to a real window too). All hachure fills get a per-shape `hachureAngle` (derived from the seed in `opts()`) so each object is filled in its own direction, like a kid's drawing. Weather is a per-game `weatherLevel` (0 clear … 3 storm) that can drift over the walk, but real rain (level ≥ 2) is capped to ~35–60s (`rainCap`) so it eases to a drizzle rather than dragging on: it adds a gloom overlay, screen-space `RAIN`, and — at storm level — a lightning `flash` plus a jagged `boltPts`, all composited in `drawWeather()` after `drawNight()`. Rain (level ≥ 2) also grows `puddles` (fixed ground spots, off the colliders) that dry out when it clears, and the cat reacts: it walks `hunch`ed, grumbles `CAT_RAIN_LINES`, and comments the moment rain begins. All are y-sorted entities drawn with `drawSprite` (which takes an optional `ent.scale` for the different dog sizes). Street lamps fade in smoothly at dusk via `lampIntensity()` (no flicker). The destination person starts `away` and arrives after a 2–3.5 min timer via one of two per-game random modes (`arrivalMode`): by tram (`tram` entity), or by coming down from the north-east panelák after occasionally peeking from its window (`peek`), then `walking → waiting` at a per-game randomized spot (`dest.tx`/`dest.ty`); the ending requires `waiting`. Worms spawn only inside the `PARK` polygon (`inPoly` test). A day-night cycle driven by `playT` (`dayPhase()`, full dark ~5 min) tints the scene toward darkness in `drawNight()`, punching light pools out of the darkness overlay for street `LAMPS` (staggered switch-on with flicker) and lit windows (`LITWINS`).
- **Input** — merged keyboard + touch joystick through `inputVec()`.
- **Update / render loop** — fixed logic in `update(dt)` (dt clamped to 0.05), rendering in `render(now)`: world blit → shadows → leash → y-sorted entities (`ents.sort` by y, painter's algorithm) → bubbles → HUD arrow → joystick → paper-grain overlay.

## Conventions

- Keep the hand-drawn aesthetic: any new visual element should be drawn through Rough.js with the shared `opts()` helper, use the `PAL` palette, get 2 seed variants for the boiling-line effect, and use deterministic seeds/jitter (no per-frame randomness in art).
- New world obstacles need a matching `block()` collider.
- New animated characters follow the existing pattern: a `draw<Name>(rc, g, seed, pose)` function, pose variants registered in `SPR`, anchor at bottom-center.
- Player-visible text stays in Czech. The cat is male (a kocour) — keep his dialogue and any text about him in masculine gender agreement.
